# 2026-02-09

## OpenClaw Cron 任务机制问题

发现 OpenClaw 定时任务存在多个 Bug：

### Bug 1: Cron "下一个"逻辑问题
- 创建时间晚于目标时间时，任务会被安排到第二天而不是立即执行
- 例如：9:05 创建的 9:00 任务，会被安排到明天 9:00 而不是今天

### Bug 2: Every 类型 + Isolated Session 不触发
- `every` 类型任务在 `isolated` session 中唤醒机制失效
- 任务状态显示正常，nextRunAtMs 计算正确，但从未实际执行
- 社区已知此问题（PR #3335 修复中）

### 解决方案
- 使用 `cron` 表达式代替 `every` 类型
- 使用 `main` session 代替 `isolated` session
- 或使用系统 cron 绕过 OpenClaw 机制
